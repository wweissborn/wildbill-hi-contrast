# Design Specifications

This document will lay out what this project is all about and why.  It is a work ivn progress and may well change over time.  

## The main topics will be:

- Scope -- What the project is and isn't
- Architecture -- How theme, build scripts and docs fit together
- Code Standards -- Formatting, Naming, file layout, that kind of stuff
- Accessibility requitements -- Readability goals, contrast targets, and testing approach
- Release process -- Versioning, git workflow, changelog practice, and packaging
- Roadmap -- Milestones from alpha to v1.0 and beyond
- Style Guide -- Explains
	- every color choice,
	- design decisions, 
	- naming conventions,
	- file organization,
	- release process,
	- version numbering,
	- accessibility goals
